# 작업지시서: 네이버 API 기반 Streamlit 트렌드 분석 대시보드 구축

본 작업지시서는 기존에 작성된 네이버 오픈 API(`쇼핑 인사이트`, `검색 API` - 블로그, 카페, 뉴스, 쇼핑) 모듈과 연동하여 **사용자가 직접 키워드를 입력하고 반응형으로 비교 분석할 수 있는 Streamlit 대시보드**를 구축하기 위한 설계 지침입니다.

---

## 1. 프로젝트 개요
- **목표**: 사용자가 동적으로 키워드를 입력하고, 해당 키워드의 최근 1년 트렌드 조회(EDA) 및 네이버 매체별(쇼핑, 블로그, 카페, 뉴스) 반응을 모니터링할 수 있는 반응형웹 대시보드 개발.
- **사용 기술**: `Streamlit` (UI 및 라우팅), `Plotly` (반응형 동적 시각화), `Pandas` (데이터 전처리), 네이버 OpenAPI.
- **선결 조건**: 이전에 구현된 `trend_analyzer/services/naver_client.py` 및 `.env` 파일의 API 통신 보안 키 매니저 재사용.

---

## 2. 세부 기능 요구사항 (Functional Requirements)

### 2.1 사이드바 메뉴 및 키워드 입력 화면
- **사이드바(Sidebar)**: 시작일/종료일 지정, 비교할 타겟 키워드 2~3개 직접 입력 가능하도록 구성.
- **카테고리 지정**: 쇼핑 인사이트 조회를 위해 대분류 카테고리(예: 패션, 스포츠, 가전 등)를 드롭다운으로 선택 기능 부여.
- **데이터 로딩 버튼**: [분석 시작] 버튼을 누르면 API 스크립트가 구동되어 캐싱(`@st.cache_data`)되도록 설계하여 불필요한 API 호출 한도 낭비 방지.

### 2.2 메인 화면 (Tab 메뉴 구조)
화면은 깔끔한 UI를 위해 Streamlit의 탭(`st.tabs`)으로 분리하여 구현합니다.
1. **기초 탐색적 데이터 분석(EDA) 탭**
2. **쇼핑 검색 결과 탭**
3. **블로그 리뷰 여론 탭**
4. **카페 활동 탭**
5. **최신 뉴스 탭**

---

## 3. 시각화 및 주요 분석 요건 (기초 EDA 탭)
이전에 정적 리포트로 만들었던 EDA 플롯을 **Plotly**를 활용하여 인터랙티브 차트로 5가지 이상 구성하며, 반드시 각각의 차트에 매칭되는 **데이터 표(Table)** 를 함께 출력해야 합니다.

1. **(차트 1) 시계열 듀얼 라인 차트**: 입력된 두 키워드의 일별 `상대 비율`을 Plotly Line 차트로 겹쳐서 표시하며 Tooltip(마우스 오버)으로 정확한 수치 표시. (표 1: 일자별 원시 데이터 테이블 동반)
2. **(차트 2) 월별 비중 비교 막대 그래프**: 키워드 간의 총 파이를 비교하는 누적/그룹형 월간 Bar 차트 구성. (표 2: 월간 피벗 테이블 동반)
3. **(차트 3) 요일별 트래픽 분포 박스플롯**: 평일과 주말의 노출 편차를 극명히 보여주는 Plotly Box 차트. (표 3: 요일별 기초 통계량 표 동반)
4. **(차트 4) 변수 간 상관관계 산점도**: 키워드 1을 X축, 키워드 2를 Y축에 둔 산점도 차트로 카니발라이제이션 유무 확인. (표 4: 상관계수 및 회귀 테이블 동반)
5. **(차트 5) 최근 30일 모멘텀 도넛 차트**: 최근 30일 간 발생한 두 키워드의 검색 총량 점유율(Market Share) 파이 차트. (표 5: 점유율 요약 표 동반)

## 4. 매체별 검색 결과 및 페이징 (검색 결과 탭)
인사이트(Trend)가 아닌, 실제 판매되고 있거나 바이럴되는 글 목록을 가져옵니다. (`/v1/search/shop.json`, `blog.json`, `cafearticle.json`, `news.json` 활용)

- **리스트 출력 규격**: 각 탭에는 검색된 아이템의 사진(지원 시), 굵은 텍스트의 **제목**, 요약 텍스트, 발행일이 출력되어야 합니다.
- **클릭 하이퍼링크**: 제목이나 URL 아이콘을 누르면 Markdown 하이퍼링크 문법(`[제목](링크)`) 혹은 `st.markdown("<a href='...' target='_blank'>", unsafe_allow_html=True)`를 통해 새 탭에서 실제 네이버 사이트로 원문 이동되도록 합니다.
- **페이징(Pagination)**: 네이버 API의 `start` 파라미터(1~1000)를 활용하여 대시보드 하단에 [이전 10건], [다음 10건] 혹은 1/2/3 페이지 버튼을 배치합니다. 한 화면에 너무 많은 스크롤이 차지 않도록 **페이지당 10~20건**씩만 렌더링되게 구성합니다.

## 5. 개발 환경 및 폴더 아키텍처

- **설치 명령어**:
  ```bash
  uv pip install streamlit plotly pandas requests python-dotenv
  ```
- **구조 제안**:
  ```text
  trend_analyzer/
  ├── dashboard.py           # Streamlit 단일 실행 엔트리
  ├── components/            # 페이징 테이블, 탭 UI 등 컴포넌트화 분리
  │   ├── charts.py          # Plotly 시각화 렌더링 함수 모음
  │   ├── search_tabs.py     # 뉴스, 쇼핑, 블로그 탭 페이징 연동 함수
  ├── config/api_env.py      # (기존 재사용)
  ├── services/naver_client.py # (기존 재사용) 검색 기능(search) 함수 추가 필요
  └── .env                   # (기존 재사용)
  ```

## 6. 최종 산출 단계 및 검수
1. `streamlit run dashboard.py` 실행 시 `localhost:8501`이 로드되는지 확인.
2. 사이드바에 키워드 '선풍기', '핫팩' 입력 후 검색 시 5개의 시각화와 표가 올바르게 나오는지(에러 로그 없는지) 검수.
3. 뉴스나 블로그 탭을 눌러 목록이 뜨는지 확인하고, 제목 클릭 시 외부 링크로 이탈이 정상 동작하는지 테스트. 페이징 버튼 클릭 시 11번~20번 데이터를 물고 오는지 최종 점검 필수.
