# 네이버 오픈API 리스트 (Open API List) 상세 통합 가이드

네이버(NAVER)는 개발자들이 자사의 플랫폼 데이터와 우수한 자체 기술들을 외부 애플리케이션이나 서비스에서 연동하여 손쉽게 활용할 수 있도록 다채로운 오픈 API와 SDK(Software Development Kit)를 제공하고 있습니다. 네이버 오픈 API 생태계는 사용자 인증을 대행하고 보호해야 하는지 여부에 따라 크게 **'로그인 방식 오픈 API'**와 **'비로그인 방식 오픈 API'** 두 가지 거대한 부류로 나뉘어 체계적으로 관리되고 있습니다. 

각 그룹이 가지고 있는 특징, 활용처, 그리고 어떤 종류의 세부 API 기술 생태계들이 존재하는지 그 기술 명세와 아키텍처를 상세히 정리합니다.

---

## 1. 로그인 방식 오픈 API (OAuth 2.0 기반 인증)

사용자 고유의 사적인 프로필 데이터에 접근하거나, 사용자를 대신하여 특정 쓰기(Write) 작업을 수행해야 할 때 의무적으로 사용하는 철저한 보안 인증 기반의 API 그룹입니다. 이 방식은 글로벌 업계 표준인 OAuth 2.0 프로토콜을 백엔드에 완벽히 구현하여 네이버 회원 로그인을 통한 "사용자 동의"를 명시적으로 얻은 후, **접근 토큰(Access Token)**을 발급받아야만 API 서버로 쿼리가 허용되는 안전한 구조를 지닙니다.

### 1.1 주요 로그인 방식 API 종류 상세 체계 안내

#### [1] 네이버 로그인 (Naver Login / Nid)
- **개요 및 아키텍처**: 외부 서드파티 서비스에서 별도의 복잡한 회원가입 절차 없이, 전 국민이 보유한 네이버 계정 인프라를 활용하여 원클릭으로 "간편 가입 및 로그인"을 실현시키는 인증 플러그인 기능입니다. SSO(Single Sign-On) 경험을 제공하여 서비스의 최초 진입 장벽을 획기적으로 낮출 수 있는 비즈니스 핵심 무기입니다.
- **주요 응용 기능**: 사용자 프로필 정보 조회(이름, 이메일 주소, 별명, 연령대 밴드, 성별, 생일 등), 사전 동의 체계(Scope) 셋팅에 따른 추가 마케팅 정보 획득 가능.
- **연동 특징 및 SDK**: 서버 사이드 REST API 뿐만 아니라, 안드로이드/iOS 전용 네이티브 SDK 컴포넌트를 제공하여 모바일에서도 네이티브 앱 UX가 끊김없이 통일되게 작동합니다. 웹사이트용 고품질 자바스크립트 버튼 SDK 및 렌더링 스크립트도 전면 지원됩니다.
- **핵심 API 연동 엔드포인트 URL**: `https://openapi.naver.com/v1/nid/me`

#### [2] 네이버 카페 (Naver Cafe)
- **개요 및 데이터 플로우**: 네이버 카페 앱이나 공식 웹 브라우저 방문 없이도, 써드파티 어플리케이션(Ex: 모바일 게임 인앱 메인 화면) 안에서 직접 네이버 카페에 가입(Join)을 신청하거나 카페 내부 게시판에 글을 올릴 수 있도록 오픈된 카페 특화 연동 API입니다.
- **주요 기능 명세**: 특정 네이버 카페 가입 신청 쿼리 발송, 사용자의 가입 여부 및 등급 확인, 게시글 목록 읽기(목록형 조회 파싱 기능), 게시판에 새로운 타겟팅 글 작성(이미지 멀티파트 파일 업로드 동반 지원). 게임 앱 내 자체 커뮤니티나 인앱 포럼을 구성할 때 생태계 확장을 위해 아주 빈번하게 쓰입니다.
- **핵심 API 연동 엔드포인트 URL 리스트**: 
  - 회원 가입 승인 요청 및 조회: `https://openapi.naver.com/v1/cafe/{clubId}/members`
  - 신규 게시글 텍스트 및 이미지 등록: `https://openapi.naver.com/v1/cafe/{clubId}/menu/{menuId}/articles`

#### [3] 네이버 캘린더 (Naver Calendar)
- **개요 및 활용론**: 외부 비즈니스 서비스에서 일어나는 티켓 예매, 호텔 숙박 예약, 비행기 출도착 스케줄 등의 주요 확정 이벤트 내역을 사용자의 개인 네이버 캘린더 계정 동기화 망에 바로 주입시켜 스마트폰 알림 파이프라인을 구축할 수 있게 하는 훌륭한 기능입니다.
- **주요 제어 기능**: 일정 생성(단순 이벤트형 시작시간/종료시간 설정, 위치 좌표 및 메모 삽입).
- **실전 킬러 활용처**: 항공권 예약 에이전시 모바일 앱, 대형 영화관 관람 예약 웹사이트, 대형 세미나 및 기술 컨퍼런스 전시회 사전 티켓 등록 플랫폼 등 리마인드가 필수적인 서비스.
- **핵심 API 연동 엔드포인트 URL**: `https://openapi.naver.com/calendar/createSchedule.json`

### 1.2 로그인 방식 API 특징적 에러 처리 방법론
서버 간 통신에서 클라이언트가 `Authorization: Bearer [Access_Token값]` 헤더를 동반하지 않거나, Access Token 내부의 타임스탬프가 만료(보통 최초 발급 후 1시간만 유효함)되었을 때 네이버 API 게이트웨이는 여지없이 `HTTP 401 Unauthorized` 또는 내부 고유 코드 `024 Authentication failed` 에러 응답 바디를 반환합니다. 
이때 클라이언트 애플리케이션의 인터셉터 로직은 리프레시 토큰(Refresh Token) 파라미터를 사용해 곧바로 서버단(Auth Server)에서 아무도 모르게 새 액세스 토큰으로 재갱신해 주는 스케줄링 로직 구조를 짜야 사용자 경험(UX) 단절을 방지할 수 있습니다.

## 2. 비로그인 방식 오픈 API (키 기반 서버 환경 인증)

사용자 개개인의 프로필이나 식별 데이터가 전혀 필요 없는 불특정 공공 데이터(검색 결과 배열, 기계 번역 매트릭스, 통계 비율 정보, AI 클라우드 예측망 등)를 고속으로 주고받을 때 작동하는 경량화 아키텍처입니다. 
네이버 애플리케이션 개발자 센터에서 발급한 **Client ID** 와 **Client Secret** 문자열 쌍을 HTTP 헤더 요소나 URI 쿼리 스트링으로 구조화하여 삽입하기만 하면 곧장 백엔드로 통신되며 결과를 반환하므로 작동 속도가 대단히 빠르고 벡엔드 아키텍처 구축이 아주 간결합니다.

### 2.1 검색 엔진 통합 API (Naver Search)
네이버가 국내외적으로 스크랩핑하여 보유하고 있는 압도적인 분량의 검색 데이터베이스 엔진을 외부망에서 쿼리하고 추출할 수 있습니다. 각 검색 도메인 섹터별로 일일 25,000회라는 매우 넉넉한 쿼리 한도를 어플리케이션당 공유하게 됩니다.

- **뉴스 전문 검색**: `https://openapi.naver.com/v1/search/news`
  - **특징 및 활용**: 특정 키워드(예: '삼성전자', '금리')와 연관된 언론사 뉴스 클리핑을 실시간 JSON 배열로 가져옵니다. `sort=sim`(유사도) 또는 `sort=date`(최신순) 옵션을 적절히 배분 파라미터로 설정해 기업 평판 관리용 최신 뉴스 스크래핑 봇이나 주식 자동 매매 정보 수집기를 만들 수 있습니다.
- **블로그 (포스트) 검색**: `https://openapi.naver.com/v1/search/blog`
  - **특징 및 활용**: 유저들의 실제 제품 사용기나 리뷰 위주의 데이터를 대용량 분석할 때 가장 좋습니다. 리뷰어의 개인 웹 URL 주소, 블로그 포스트 작성 날짜 등의 메타 데이터를 XML 혹은 JSON으로 깔끔하게 디코딩해서 송신합니다.
- **지역 점포 (Local) 위치 검색**: `https://openapi.naver.com/v1/search/local`
  - **특징 및 활용**: 검색어에 기반하여 특정 지역의 맛집 상호명, 주소 정규화 데이터, 지도 상의 건물 위치 좌표 등을 뽑을 때 탁월합니다. 좌표계는 KATECH 방식을 기본으로 응답이 리턴됩니다.
- **쇼핑 상품 카이로스 검색**: `https://openapi.naver.com/v1/search/shop`
  - **특징 및 활용**: 네이버 쇼핑몰 DB에 파트너들이 등록한 최저가 랭킹 데이터 및 상품 카탈로그 정보 열람이 구조화되어 반환 가능해 최저가 가격 비교 서비스나 무인 상점 가격표 제작에 필수적입니다.
- **웹문서 및 백과사전 지식 쿼리**: `../search/webkr`, `../search/encyc`
  - **특징 및 활용**: 지식 레퍼런스를 인용 및 참조해야 하는 학습 도우미 서비스 환경이나, 챗봇 사전 지식 공급 용도(RAG)에서 주로 백엔드 파이프라인으로 호출됩니다.

### 2.2 비즈니스 데이터 플랫폼 (DataLab) API
현대 디지털 마케팅 상업 분야에서 대단히 귀중한 소비자 빅데이터 관심 수요 트렌드 곡선 수치를 비율화하여 도출해 주는 거시적 통계용 프로바이더입니다.
- **데이터랩 - 통합검색어 추이 변화량**: `https://openapi.naver.com/v1/datalab/search`
  - 10대, 20대 등 타겟 인구통계학적 집단의 특정 기간별 검색어 관심도를 상대 비율(최대 100) 지표로 정교하게 계산하여 제공합니다.
- **데이터랩 - 쇼핑 인사이트 세분류**: `https://openapi.naver.com/v1/datalab/shopping/...` 카테고리
  - 특정 쇼핑 세부 카테고리 분류별, 기기 환경별(PC/MO), 연령 모델별, 성별 클릭율 전환 추이 지표를 추출해 타겟층 분석에 절대적 무기를 제공합니다.

### 2.3 기계 지능 및 언어 처리 통합 API (Papago, Clova Cloud)
현대 AI 기술이 깊숙하게 접목되어 방대한 GPU 인프라 구조의 인공 신경망 모델을 이용해 동작하는 강력한 클라우드 툴킷 시리즈입니다.
- **Clova Face Recognition (CFR 얼굴 인지 시스템)**: `https://openapi.naver.com/v1/vision/face`
  - 클라이언트 유저가 백엔드로 올린 인물 멀티미디어 사진 이미지를 머신러닝 AI 분석하여 프레임 속 인물 나이대 추정 구간치, 성별 확률 분배, 화남/웃음/슬픔 등의 복잡한 안면 감정 상태를 정밀하게 파악해 줍니다. 
- **Clova Celebrity (유명인 얼굴 형상 인식)**: `https://openapi.naver.com/v1/vision/celebrity`
  - 사진과 가장 높은 벡터 싱크로율로 닮은 대중문화 유명인의 매핑 이름과 확신도 매칭 퍼센티지(Confidence Level)를 정확도별 배열로 반환합니다. MZ세대 타겟의 게임/예능 엔터테인먼트 마케팅 용도로 빈번히 채택되어 활용됩니다.
- **파파고 (Papago) NMT 인공신경망 기계 번역**: (참고: 현재 네이버 오픈 API 생태계를 넘어 네이버 클라우드 B2B 플랫폼으로 대거 이전되어 심화 운영됨)
  - 다국어 AI 자동 번역 심층 기술입니다. 문맥적 의미망을 기반으로 한 인공신경망 엔진(NMT)을 사용하여 한국어/영어/일본어/중국어 언어쌍의 상호 변환의 스무스한 자연스러움을 인간 번역가 수준으로 극대화 해냅니다.
- **파파고 텍스트 언어 감지 예측기**: 사용자가 어떠한 언어로 입력한 텍스트 덩어리의 종류(국적)가 실제로 무엇인지 언어학적 패턴으로 판별 및 국가 표준화된 코드 세트 산출(`ko`, `ja`, `en`, `zh-CN` 등)해 주는 기반 API를 다룹니다.

### 2.4 네이버 지도 (Maps API) 아키텍처
모빌리티 및 공간 정보 솔루션을 지향하는 플랫폼에 있어 가장 강력한 무기입니다. (마찬가지로 네이버 클라우드 플랫폼 기반으로 서비스 통합 운영 중)
- **Web Dynamic Map**: HTML5 Canvas 기반의 부드러운 벡터 동적 지도 렌더링.
- **Geocoding / Reverse Geocoding**: 지번 주소를 KATECH/WGS84 위경도 좌표로 완벽하게 치환하거나, 위경도 좌표를 보내면 행정동 주소 문자열을 파싱해 주는 공간 지능화 서비스입니다.
- **Directions (자동차 길찾기 및 내비게이터)**: 출발지와 도착지 파라미터를 주입하면 최적의 네비게이션 주행 경로 노드 배열을 반환합니다.

### 2.5 유틸리티 및 웹 보안 보조 기능 (Captcha / Share)
- **보안 캡차 머신 (Image Captcha Generator 시스템)**
  - 네이버 본사 웹서비스 회원가입에서 사용되는 최신 이미지 캡차 왜곡 문자를 외부 API로도 똑같이 생성 및 검증하는 인터페이스 생태계. 트래픽 어뷰징 공격(DDoS, 가입 봇 머신) 방지에 탁월한 가성비 솔루션입니다. 구조상 먼저 `v1/captcha/nkey`부터 통신을 전송하여 난수 암호 키 셋상을 발급받아야 다음 동작이 이루어지는 고도화 구조입니다.
- **오픈메인 / 공유하기 외부 뷰어 공유 인터페이스**
  - 자사의 뉴스 기사 글이나 블로그 콘텐츠를 네이버 밴드 쉐어 허브, 네이버 블로그 스마트에디터 작성 창으로 곧장 넘겨버리는 웹 브라우저 DOM 기반 오픈 플러그인. 통계용 프록시 서버를 따로 구축하지 않고 바로 전송되게 자바스크립트 수준에서 스크립트 연결만으로 연동되므로 구축비용이 0원에 수렴합니다.

## 3. 핵심 요약 및 보안 지향적 아키텍처 권장 사항

1. 네이버의 **로그인 방식 API** (Oauth) 체계는 외부 고객 개인 데이터의 프라이버시 보호를 위해 매우 엄격한 라이프사이클 프로토콜이 적용되므로, 메인 백엔드 서버(Spring Boot, Node/Express, Python Django) 상에서의 세션 캐싱 및 토큰 플로우 설계가 치밀하게 기획되어야 합니다.
2. 반면에 **비로그인 방식 API** (데이터랩, 로컬 검색, 클로바 등)는 React나 iOS/Android 클라이언트 어플리케이션 소스 코드 안에서 네이버로 직접적으로 API 후크(호출)를 날려 버리면, 패킷 캡처 와이어샤크나 디컴파일 유틸리티 등에 의해 소중한 `Client Secret` 값이 유출 및 분실될 엔드포인트 우려가 아주 농후합니다. 
3. 따라서 언제나 **자사 도메인을 가진 중간 API 프록시 계층 레이어**를 안전판으로 두고, 프론트단(React, 모바일앱)에서는 브라우저 상의 해당 자사 서버 주소로만 통신을 날려 프론트엔드 환경 쪽에 네이버 시크릿 암호키가 일체 노출되지 않도록 하는 **BFF (Backend For Frontend)** 라우팅 방화벽 아키텍처 패턴이 전사적 개발 표준 가이드라인으로 강하게 업계 내 권고되고 있습니다.
4. 모든 종류의 비로그인 API를 테스트 및 파일럿팅해 볼 때, 우선 Linux Bash 콘솔 환경에서 간단한 `cURL -H` 통신 명령어 테스트부터 여러차례 시행하여 네트워크 방화벽 아웃바운드 개방 체크와 상태 코드 `200`이 무사히 반환됨을 CLI에서 목격한 후 본 자바나 함수 코딩에 돌입하는 것이 API 응답 JSON 형식을 디버깅하고 스키마를 맵핑하는 가장 에러가 적고 효율적인 개발론 프로세스입니다.
